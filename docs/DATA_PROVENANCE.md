Every CSV the Lab reads is committed so the backtest reproduces with no
network and no database. Original sources, spans, and regeneration paths:

| File | Series | Original source / venue | Span | Regenerate |
|---|---|---|---|---|

| `data/lab/ls_bybit.csv` | BTCUSDT long/short-ratio snapshots | Bybit public API (same relay) | 2025-04-03 → 2026-06-10 | `python -m lab.export` |

| `data/lab/liq_events.csv` | BTCUSDT liquidation events | Bybit public API (same relay) | 2026-05-06 → 2026-06-10 | `python -m lab.export` |
| `data/lab/bands_rm17.csv` | rm17-derivs Band labels (`regime_matrix_labels`, run `rm17-derivs-4h-20260530`) | internal lab derivation over the series above (not a third-party feed) | 2025-10-04 → 2026-05-18 | `python -m lab.export` |

All commands run under `uv run --no-sync`.

These files are small derived extracts of the providers' data, redistributed
solely for research reproducibility — so judges and readers can re-run the
pre-registered sweep without our infrastructure. All rights to the underlying
data remain with the respective providers (Binance, Bybit, Deribit,
CoinMarketCap). The one CoinGlass-relayed series we consulted was used ONLY
for the funding cross-check and is not committed as a dataset; we intentionally
document its 2025-04-08 storage-convention switch
(`data/backfill/funding_crosscheck.txt`) as data-integrity evidence — it is
exactly the mid-window source hazard the feature source-consistency rule
(R-SRC) exists to catch, and it is why the Lab uses Binance REST funding
end-to-end.
